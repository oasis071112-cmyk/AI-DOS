# Brand Tone

## Purpose

Brand Tone defines how AI-DOS communicates through language.

Every piece of text should be clear, professional, and helpful.

---

## Tone Characteristics

The default tone should be:

- Clear
- Friendly
- Professional
- Calm
- Direct
- Confident

Never sound robotic or overly promotional.

---

## Writing Principles

Good writing should:

- Be easy to read.
- Be concise.
- Focus on the user's goal.
- Reduce cognitive load.
- Encourage action.

Prefer short sentences over long explanations.

---

## Buttons

Use action-oriented labels.

Examples:

- Create Project
- Save Changes
- Continue
- Learn More
- Start Building
- View Details

Avoid:

- Submit
- Click Here
- Confirm
- OK

---

## Error Messages

Every error message should answer three questions:

1. What happened?
2. Why did it happen?
3. What should the user do next?

Example:

Unable to upload the file.

The file exceeds the maximum size limit.

Please choose a file smaller than 10 MB and try again.

---

## Success Messages

Success messages should be short and positive.

Examples:

- Project created successfully.
- Changes saved.
- Settings updated.
- Invitation sent.

Avoid unnecessary excitement.

---

## Empty States

Every empty state should explain:

- What this page is.
- Why it is empty.
- What the user can do next.

Example:

No projects yet.

Create your first project to start building with AI-DOS.

---

## Confirmation Dialogs

Explain:

- What action is about to happen.
- Whether it can be undone.
- What the user should expect.

Avoid vague confirmation messages.

---

## AI Rule

Every sentence should help users understand, decide, or act.

If a sentence does not add value, remove it.
